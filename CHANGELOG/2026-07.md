# Changelog — 2026-07

### 2026-07-01 — fix: bake the welded command plugins into the .deb (arch parity)

The welded command plugins (`command:<word>` for doctor/clean/settings/candy/tmux/
preempt/feature/vm/secrets/udev) are out-of-process and resolve ONLY from
`bakedPluginDirs()` = `$CHARLY_PLUGIN_DIR` + `/usr/lib/charly/plugins`. The arch
PKGBUILD bakes them beside the binary; this package installed ONLY `charly usr/bin`,
so on a debian `apt install ./opencharly-*.deb` `charly doctor`/`clean`/`vm`/… were
unresolvable project-less ("unknown command").

- `debian/opencharly.install` now also installs `plugins/* usr/lib/charly/plugins`
  (the host-built welded plugins the deb `local_pkg.build_template` copies into the
  package source tree as `plugins/`).
- `debian/rules` adds `override_dh_fixperms` to re-assert 0755 on the plugin binaries
  after `dh_fixperms` (which resets `/usr/lib` regular files to 0644); the `.providers`
  word manifests stay 0644.
- The plugin SET is the shared single source `pkg/host-command-plugins.txt` in the
  superproject (R3) — the SAME list the arch PKGBUILD and the fedora `.rpm` build read.

Proof (R8): built the deb via the modified `build_template` (podman debian:13
dpkg-buildpackage) — `dpkg -c` shows all 10 `plugin-*` (0755) + `.providers` (0644)
under `/usr/lib/charly/plugins`; `dpkg -i` + a project-less `charly doctor` dispatches
the baked `command:doctor` plugin. Full welded-command `charly check run` gate:
`check-debian-debootstrap-vm` (box/debian) gained a `welded-command-project-less`
check step.
